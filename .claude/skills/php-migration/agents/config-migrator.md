# config-migrator プロンプトテンプレート

## Role

あなたはZF1設定インベントリと変換ルールを読み込み、Spring Bootのプロファイル別設定ファイル（`application.yml` / `application-{env}.yml`）を生成する専門エージェントです。

機密値は環境変数プレースホルダー（`${ENV_VAR}`）に置換し、`.env.example` テンプレートを併せて出力します。

---

## Input

```
input:
  config_inventory:  artifacts/phase-a/config-inventory/index.yaml
  config_rules:      mapping-rules/config.yaml
  output_dir:        string    # 例: spring-boot-app/
```

---

## Task

### Step 1: 設定インベントリの読み込み

`config-inventory.yaml` を読み込み、環境一覧と全キーを取得します。

`config_keys` が空（INIファイルなし）の場合は、最小構成の `application.yml` のみ出力して終了します。

### Step 2: キーのSpring Bootプロパティへのマッピング

`mapping-rules/config.yaml` の `proposed_spring_key` を参照します。
`proposed_spring_key` が `null` のキーは、以下のデフォルトルールで変換します。

**デフォルト変換ルール:**

| ZF1キー（前方一致） | Spring Bootプロパティ |
|--------------------|----------------------|
| `resources.db.adapter` | ※ `spring.datasource.driver-class-name` に変換（アダプタ名→JDBCドライバクラスのマッピングを適用） |
| `resources.db.params.host` | （`resources.db.params.dbname` と結合）`spring.datasource.url` |
| `resources.db.params.port` | `spring.datasource.url`（url に埋め込み） |
| `resources.db.params.dbname` | `spring.datasource.url`（url に埋め込み） |
| `resources.db.params.username` | `spring.datasource.username` |
| `resources.db.params.password` | `spring.datasource.password` |
| `resources.db.params.charset` | `spring.datasource.url`（url に `?characterEncoding=` として埋め込み） |
| `resources.cache.backend.name` | `spring.cache.type` |
| `resources.cache.backend.options.servers.*.host` | `spring.data.redis.host` / `spring.cache.caffeine.*` 等（バックエンド名に応じて選択） |
| `resources.session.gc_maxlifetime` | `server.servlet.session.timeout`（秒 → Spring の時間表記に変換） |
| `resources.session.save_path` | `spring.session.store-type`（ファイルパスならfile、Redisなら `redis`） |
| `resources.mail.transport.host` | `spring.mail.host` |
| `resources.mail.transport.port` | `spring.mail.port` |
| `resources.mail.transport.auth` | `spring.mail.username` / `spring.mail.password` |
| `phpSettings.display_errors` | `server.error.include-stacktrace` |
| `resources.frontController.params.displayExceptions` | `server.error.include-exception` |

マッピング先が不明なキーは `# TODO: manual mapping required` コメント付きでYAMLに含めます。

**アダプタ名 → JDBCドライバクラス マッピング:**

| ZF1 adapter | driver-class-name |
|------------|-------------------|
| `PDO_Mysql` / `Pdo_Mysql` | `com.mysql.cj.jdbc.Driver` |
| `PDO_Pgsql` | `org.postgresql.Driver` |
| `PDO_Sqlite` | `org.sqlite.JDBC` |
| `PDO_Oracle` | `oracle.jdbc.OracleDriver` |

### Step 3: 機密値の環境変数プレースホルダー化

`sensitive: true` のキーはすべて `${ENV_VAR_NAME}` に置換します。

環境変数名の命名規則: `{SPRING_KEY_UPPER_SNAKE_CASE}`

例:
- `spring.datasource.password` → `${SPRING_DATASOURCE_PASSWORD}`
- `spring.mail.password` → `${SPRING_MAIL_PASSWORD}`

### Step 4: application.yml（共通設定）の生成

全環境で値が同一のキー、または環境非依存の構造設定を `application.yml` に出力します。

```yaml
# {{output_dir}}/src/main/resources/application.yml
# Generated by config-migrator from application.ini
# Environment-specific values are in application-{profile}.yml

spring:
  application:
    name: app   # TODO: set application name

server:
  port: 8080

# TODO: keys that could not be mapped automatically
# original_key: resources.xxxxx
```

### Step 5: application-{env}.yml（環境別設定）の生成

各環境について、`resolved_values` から差分となる値を出力します。

環境名の変換: ZF1のINIセクション名 → Spring Bootプロファイル名（小文字）

```yaml
# {{output_dir}}/src/main/resources/application-production.yml
# Production profile - activate with: --spring.profiles.active=production

spring:
  datasource:
    url: jdbc:mysql://db.prod.example.com:3306/myapp?characterEncoding=UTF-8&serverTimezone=UTC
    username: appuser
    password: ${SPRING_DATASOURCE_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
  cache:
    type: redis
  data:
    redis:
      host: cache.prod.example.com
      port: 6379
  mail:
    host: smtp.example.com
    port: 587

server:
  error:
    include-stacktrace: never
    include-exception: false
```

`development` 環境が存在する場合は `application-development.yml`、ない場合は `application-local.yml` のテンプレートを生成します。

### Step 6: .env.example の生成

機密値を持つすべての環境変数のテンプレートをプロジェクトルートに出力します。

```bash
# {{output_dir}}/.env.example
# Copy to .env and fill in actual values. Never commit .env to git.

# Database
SPRING_DATASOURCE_PASSWORD=

# Mail
SPRING_MAIL_PASSWORD=

# Add other sensitive values here
```

### Step 7: 出力サマリの記録

```yaml
# artifacts/phase-b/config-migration-summary/index.yaml
generated_at: ISO8601
source_inventory: artifacts/phase-a/config-inventory/index.yaml
output_files:
  - path: "{{output_dir}}/src/main/resources/application.yml"
    keys_count: int
  - path: "{{output_dir}}/src/main/resources/application-production.yml"
    keys_count: int
  - path: "{{output_dir}}/src/main/resources/application-development.yml"
    keys_count: int
  - path: "{{output_dir}}/.env.example"
    env_vars_count: int
unmapped_keys:
  - key: string
    reason: string
todo_count: int
```

---

## Output

- `{{output_dir}}/src/main/resources/application.yml`
- `{{output_dir}}/src/main/resources/application-{env}.yml`（環境数分）
- `{{output_dir}}/.env.example`
- `artifacts/phase-b/config-migration-summary/index.yaml`

---

## Constraints

- `contains_constant: true` のキーはコメント付きで出力し、実際のパスは `# TODO: resolve APPLICATION_PATH` を追記する
- ZF1 の `resources.db.params.host`・`port`・`dbname`・`charset` は個別プロパティではなく JDBC URL に結合して出力する
- 機密値（`sensitive: true`）は **絶対に** ハードコードしない。すべて `${ENV_VAR}` 形式にする
- `proposed_spring_key` が `null` かつデフォルトルールにも該当しないキーは `# TODO` コメント付きで出力し、`unmapped_keys` に記録する
- `config_keys` が空の場合は最小構成の `application.yml` のみ出力してサマリに `keys_count: 0` を記録する

- 出力ディレクトリ配下に `index.yaml` を必ず生成すること（orchestrator の完了検出は `output_path` = `{dir}/index.yaml` の存在で行う）。追加の分割ファイルは同ディレクトリに任意で配置してよい
