## 着手優先順序

現時点では Phase A/B/C の基本動作は確認済み（study-sa での実行テスト完了）。  
次に整備すべき順序：

1. **3-4 DoD チェックリスト** — Phase C の「完了」判定を機械化するために先に定義が必要
2. **3-2 タスク仕様フォーマット** — タスクを構造化することで Phase C の再現性が上がる
3. **4-1/4-2 コンテキストパック** — 大規模プロジェクトへの適用時にトークン効率が課題になる
4. **3-3 実装結果レポート** — 実行履歴の蓄積・振り返りに必要
5. **5-5/5-6 設計書・拡張ガイド** — Spring Boot 等への対応拡張時に必要
